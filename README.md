# Socratic

Socratic 是一个面向成年人的 AI 学习教练产品设想：不让用户重新坐回课堂，而是每天通过一个问题，重新训练数学、物理和科学思考能力。

## 背景

AI 时代会让确定性任务越来越自动化，人类更需要持续学习、迁移认知、理解复杂世界。但很多成年人已经脱离学生时代，很难再从教材第一章开始系统学习。

问题不在于缺少课程或资料，而在于：

- 不知道从哪里重新开始。
- 对庞大的学习量有畏惧。
- 缺少即时反馈和陪伴。
- 一遇到抽象概念就容易放弃。
- 传统学习方式容易唤起考试、排名和挫败感。

本项目希望探索一种更轻、更自然的学习入口。

## 产品定位

每天一个问题，重新训练你的科学直觉。

这是一个 AI 对话产品，但它不是普通问答机器人，也不是传统课程平台。它更像一个克制的苏格拉底式教练：

- 不急着给答案。
- 鼓励用户先表达直觉。
- 通过追问暴露知识缺口。
- 再引入最小必要知识点。
- 最后用迁移问题确认用户是否真正理解。

前台是轻量对话，后台是知识图谱、能力诊断和个性化教学路径。

## 当前落地形态

当前先把产品落成一个平台无关的通用 Skill 包：

`/skill/`

它不是底层 Agent Runtime，也不负责通知、调度、长期记忆存储或用户会话通道。这些能力应由 OpenClaw 这类主动式 agent 承担。

Skill 负责上层应用智能：

- 从用户表达中澄清学习意图。
- 构建或扩展知识图谱。
- 基于学习状态规划下一课。
- 支持主动推送和用户主动学习两种入口。
- 用苏格拉底式对话引导用户思考。
- 在学习结束后生成结构化状态更新。

## 核心体验

用户每天或不定期收到一道题。题目会根据用户当前知识水平调整抽象程度：

- 知识水平较低时，题目尽可能贴近日常生活。
- 知识水平较高时，题目可以更抽象、更形式化。
- 用户不需要一开始就计算，而是先提出解题思路。
- AI 在对话中保持克制，用提示和追问引导用户自己思考。
- 当问题自然延伸出知识点时，AI 再进行短教学。
- 系统记录用户掌握、误解和卡住的概念，并构造下一步学习路径。

## 学习闭环

每一次学习都应该形成一个小闭环：

1. 推送一个问题。
2. 用户先给出直觉或猜想。
3. AI 追问用户为什么这么想。
4. AI 用分层提示帮助用户推进。
5. 暴露一个关键知识缺口。
6. 讲解解决该问题所需的最小知识点。
7. 给出迁移题，确认用户是否能够换场景使用该概念。
8. 更新用户知识状态。

## 示例问题

### 低抽象程度

为什么冬天摸铁门把手，会感觉比摸木头桌子更冷？它们明明在同一个房间里。

可延伸知识点：

- 温度
- 热量传递
- 热传导率
- 材料属性
- 人体感知与物理量的区别

### 中等抽象程度

为什么骑车下坡时，刚开始速度增加不明显，后来会越来越快？

可延伸知识点：

- 速度
- 加速度
- 力
- 能量转化
- 变化率

### 高抽象程度

如果一个量的变化速度本身也在变化，我们应该如何描述这种变化？

可延伸知识点：

- 函数
- 导数
- 二阶导数
- 局部变化
- 数学建模

## AI 行为原则

AI 的克制程度是这个产品的核心。

- 用户第一次尝试前，不直接给标准答案。
- 用户答错时，不直接羞辱或否定，而是构造反例。
- 提示分层释放：轻提示、方向提示、关键提示、完整解释。
- 每次只引入一个核心新概念。
- 解释之后要求用户用自己的话复述。
- 用户焦虑时，降低抽象程度，而不是继续堆知识。
- 不把学习包装成考试、排名或惩罚。

## MVP 范围

第一版可以只做一个 30 天的实验产品：

- 每天一道数学或物理直觉题。
- 每题控制在 10 到 15 分钟。
- 支持用户用自然语言和 AI 对话。
- AI 通过追问和提示引导用户。
- 每天结束后生成一段学习复盘。
- 记录用户点亮过的概念。

第一阶段主题建议：

- 变化：速度、增长、斜率、导数。
- 空间：方向、向量、坐标、距离。
- 力：推、拉、摩擦、平衡。
- 能量：做功、热、效率。
- 概率：不确定性、期望、风险。
- 模型：如何把现实问题变成数学问题。

## 设计原则

- 先问题，后概念。
- 先直觉，后公式。
- 先解释，后计算。
- 先安全感，后挑战。
- 前台轻量，后台系统。
- 不制造学习焦虑。
- 不让 AI 替用户思考。

## 暂定技术方向

技术栈暂不锁定。后续可以根据验证方式选择：

- Web 原型：React / Next.js
- 本地快速实验：Vite / React
- 对话状态管理：轻量状态机或服务端 session
- 知识图谱：JSON / SQLite / Postgres
- AI 能力：LLM API + 提示协议 + 教学策略层
